Splash Screen

Technical Description

A temporary landing page called splash screen offers website intro services before using an automatic redirect to send users to the home page after 4 seconds. It features: The smooth transition is achieved through the implementation of "meta http-equiv="refresh" content="4; url=home.html". The .splash-container contains a semi-transparent black background for clear text presentation which overlays images/img2.jpg. Vertical and horizontal element centering through flexbox creates a clean professional layout for this page. The Animated Loader contains a div that spins through CSS keyframes animation (@keyframes spin) which provides users with a visual waiting indication before page redirection. The Page Editor button uses CSS styling to function as a navigation tool which redirects users directly to home.html. The site organization automatically adjusts its display for different screen sizes by using a combination of adaptable widths and text alignment.


Accessibility

1. Semantic HTML Structure:

The HTML code applies meaningful elements such as h1, p and ul to support screen reader compatibility and improve document structure.

2. Alt Text for Images:

Screen reader accessibility requires alt attributes with the empty quotation marks (alt="") for every image-based logo or additional icons present in the design.

3. Keyboard Navigation:

Users can navigate to the “Page Editor” button (.editor-button) by using Tab key keyboard navigation and activate it with Enter key. Users with motor disabilities can better see the button through its hover effect on top of its keyboard accessibility.

4. Auto-Redirection Considerations:

Users who depend on assistive technology may have difficulty with sudden redirections caused by the meta refresh element meta http-equiv="refresh" content="4; url=home.html". The better practice includes enabling manual navigation through an alert that warns of redirection.

Volunteer Page

Technical Description

The HTML document of the volunteer page shows different volunteer opportunities through a well-organized format. The webpage functions through the fusion of HTML and CSS and JavaScript to supply dynamic user-friendly feature This page exhibits a header section that contains page title along with logo and navigation menu. The page contains main content which resides within an element called div class="container". Multiple div class="card" elements for displaying volunteer opportunities. A footer section with additional links and credits. Several external CSS files (stylesheet.css, global3.css) are linked in the document. The additional styles are included inside an inline style block placed within the head element. The page makes use of flexbox while using grid components to achieve layout adjustments based on device size. The website features hover effects that apply to select buttons as well as dropdowns and cards. The application features a review system which enables users to reveal more content through buttons for expanding and collapsing. Additional reviews appear or disappear through the "See More Reviews" button. The application uses event listener functions to control user-triggered element expansions and population changes. Uses Font Awesome (https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.2/css/all.min.css) for icons. The content includes additional information and volunteer program links that users can access.


Accessibility

1. Semantic HTML Structure:

Uses h1, nav, section, footer for better readability by screen readers.

2. Keyboard Navigation:

Each button and dropdown menu functions when tab is pressed.

3. Color Contrast:

Many parts of the text remain readable against backgrounds yet there are some areas which require improvement.

Content Page

Technical Description

The structured HTML document of the content page serves as a platform for educational information regarding physical fitness. The implementation of HTML and CSS and minimal JavaScript serves to deliver a quality user experience with proper presentation of information. The header section includes a logo area with title along with its navigation menu. A sidebar navigation ul id="sideBar" for quick access to different sections. The main content consists of a structured div area which contains text content through paragraphs and headings from h2 to h3. The page contains several img elements which have the "center-image" class for image presentation purposes. A footer section with additional links and credits. External Stylesheets: Links global3.css and content_page.css for layout consistency. Background and Font Styles: Uses Arial, sans-serif for readability. All content faces left and right borders to achieve an official justification effect. The navigation and header sections receive designs using certain background attributes and vertical spacing elements. To align images at their centers the class .center-image is utilized. Users can access various sections through the ul id="sideBar" section which contains direct anchors. Users can easily navigate through content content while it stays inside a fixed area. Currently accessible viewing is possible through images that integrate alt attributes with their code.


Accessibility

1. Alternative text:

Alternative Text (alt) provides images an accessible description for users who depend on screen reading software.

2. Good color contrast:

The color contrast between elements stands at an acceptable level for reading purposes.

Challenges and Lessons Learned

1. Ensuring Accessibility for All Users

  • challenges: Everyone from disability users to non-disabled users must have clear navigation routes to comprehend the content homogeneously.
  • Lessons learned: Ensuring keyboard navigation improves accessibility. Users with visual impairments benefit from proper color contrast selection because it improves their ability to read the content.

2. Maintaining Consistency in Design & Layout

  • challenges: The application required a uniform design layout because global3.css and content_page.css were both utilized as stylesheet files.
  • Lessons learned: A common CSS file with reusable design rules should be created for buttons and headers along with cards to establish design consistency.

Compliance

Security & Data Privacy (GDPR Compliance):

User consent must be obtained before collecting any personal data from forms.
No use of cookies or trackers without explicit notification.
Web pages containing external links display content in new windows while using "rel="noopener noreferrer"" to safeguard against possible security risks.

Accessibility Compliance (WCAG 2.1 Standards):

Pages receive enhanced accessibility because they implement semantic HTML elements including h1, h2 and nav and footer.
Users who have visual impairments can access images effectively when developers add proper Alt attributes to them.
Readability is ensured by considering color contrast to some degree.
The system allows keyboard operation users to move between links and buttons by pressing the Tab key.
Pages support text resize effectiveness using em and percentage units as well as rem units.
