Team Page

Technical Description

The file is a Team Page for a website focused on "Good Health and Well-being." The team page (team.html) employs semantic HTML5 elements with a proper document structure, including a header, navigation, main content area, and footer. It introduces the project team members and includes navigation, images, and structured content.


Accessibility

1. Semantic HTML Structure:

The page uses proper semantic elements like <header>, <nav>, <main>, <section> and <footer> to create a logical document structure that screen readers can navigate effectively. This also provides main page navigation using links and also organizes team member profiles that represents each team member.

2. Alt Text for Images:

All images, including the profile pictures of each team member, the background image and the logo icons, have descriptive alt attributes that provide context for screen reader users and it also provides a decent look and feel.
  • Team Member Profile Picture(raphael-pfp.png, etc...)
  • Background Image
  • Fallback Image(<onerror="this.src='images/pfp.png';">)

3. Keyboard Navigation:

The layout and interactive elements are designed to be fully navigable using keyboard controls, with proper focus management for links and form elements. Also uses aria-label for better screen reader descriptions.

4. Visual Hierarchy:

Clear heading structure and consistent styling help users with cognitive disabilities understand the content organization. It also has navigation bar with clearly spaced links, bold descriptions and structured profiles, textal descriptions using list bullets for readability and multiple sections using headings for better clarity.

Feedback Page

Technical Description

The Feedback Page allows users to provide their feedback on the website. It includes a form with input fields for name, email, feedback and drop down for user rating. It also includes a button to submit the feedback.


Accessibility

1. Semantic HTML Structure:

The document follows a semantic HTML structure consistinga website logo,a website title, a navigation bar, a main content area, and a footer. The main content area includes a form, input fields, and a button also including a background image in this page.

2. ARIA Labels:

The page contains a form where users can submit their opinion on the website which improves accessibility. <aria-labels>should be used specifically to assist the user's reading.

3. Visual Hierarchy:

<h1>used for website title and <h2>used for Feedback Form title. The form is structured using <label> and <input> elements. In this page the form labels are associated with the input fields and also includes placeholders for better user experience. A single form consists of <value> where it stores a temporary value.

Content Page

Technical Description

The HTML page uses semantic elements like <header>, <nav>, <section>, and <footer> to create a logical structure for health awareness content. The CSS employs fixed positioning for the sidebar navigation, ensuring consistent access to content sections. The main content area uses overflow scrolling to handle extensive text. The image gallery utilizes CSS floats and borders for responsive layout, while tables employ <border-collapse> for readability. Key CSS properties include <margiin>, <pading>, and hover effects to enhance usability and visual feedback. The design balances fixed elements for navigation with fluid content areas, creating a user-friendly experience that presents complex nutritional information in an accessible format.


Accessibility

1. Meaningful navigation:

The navigation menu employs descriptive link text and an unordered list format, which is accessible to assistive technologies. This also makes the navigation menu easier to navigate for users and to switch into sepcific topics fastwhile reading.

2. Better image alignment:

The images are well used and the positions are well fixed using image alignment. The images are under a <div> element where there is a class used for the <div> element <div class=imgs>which consists of image alignment except individual image alignment as well.

Challenges and Lessons Learned

1. It was diificult to decorate images and it was even more difficult to fix the size of them.

2. When creating the glow over hover effect in the Team Page, it was diificult to make it look more brighter as I had to use several box-shadow statements to get it right.

3. It was also difficult to make the background image in Team Page and Feedback Page look less brighter and it was even more difficult to fix the size of them while making it not repeat when the user scrolls.

4. It was comparatively difficult to make the images have an alternate image if the main image is gone or missing.

5. I learnt that giving an alternative text and an alternative image for an image is very important and also a better practice.
