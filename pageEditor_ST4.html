User Profile Page

Technical Description

The user profile page (profile.html) employs semantic HTML5 elements with a proper document structure, including <meta> tags for responsiveness and accessibility. The layout utilizes CSS flexbox through .container and .right-and-calendar classes to create a responsive design with a <div class="left-column"> displaying profile information and a <div class="right-section">containing about me, goals, frustrations, experience, and contact form sections. The contact form implements HTML5 validation using <input required> and <textarea required> while the schedule table uses alternating row colors via CSS for readability. CSS custom properties like <--primary> and <--neutral-lighty> ensure consistent theming, and hover effects on .btn and navigation links enhance interactivity. Accessibility is addressed through semantic markup, alt attributes for <img> elements, and sufficient color contrast between text and background elements.


Accessibility

1. Semantic HTML Structure:

The page uses proper semantic elements like <header>, <nav>, <main>, and <footer> to create a logical document structure that screen readers can navigate effectively.

2. Alt Text for Images:

All images, including the profile picture and UN goal icons, have descriptive alt attributes that provide context for screen reader users.

3. Keyboard Navigation:

The layout and interactive elements are designed to be fully navigable using keyboard controls, with proper focus management for links and form elements.

4. Visual Hierarchy:

Clear heading structure and consistent styling help users with cognitive disabilities understand the content organization.

Sitemap Page

Technical Description

The sitemap page (sitemap.html) utilizes semantic HTML5 with a proper document structure, including <meta> tags for responsiveness. The layout employs an <svg> element to create an interactive visual representation of the site structure, with <rect> elements for nodes, <text> for labels, and <line> for connections. CSS flexbox and media queries ensure responsive behavior, while custom properties like <--primary> maintain consistent theming. Accessibility is addressed through ARIA labels, sufficient color contrast, and keyboard-navigable elements.


Accessibility

1. Semantic HTML Structure:

The page uses proper semantic elements like <header>, <nav>, <main>, and <footer> to create a logical document structure that screen readers can navigate effectively.

2. ARIA Labels:

The SVG elements include aria-label attributes that describe each node and connection, ensuring screen readers can interpret the diagram.

3. Color Contrast:

Sufficient contrast ratios between elements ensure readability for users with visual impairments, following WCAG guidelines.

Content Page

Technical Description

The HTML page uses semantic elements like <header>, <nav>, <section>, and <footer> to create a logical structure for nutrition content. The CSS employs fixed positioning for the sidebar navigation, ensuring consistent access to content sections. The main content area uses overflow scrolling to handle extensive text. The image gallery utilizes CSS floats and borders for responsive layout, while tables employ <border-collapse> for readability. Key CSS properties include <margiin>, <pading>, and hover effects to enhance usability and visual feedback. The design balances fixed elements for navigation with fluid content areas, creating a user-friendly experience that presents complex nutritional information in an accessible format.


Accessibility

1. Meaningful navigation:

The navigation menu employs descriptive link text and an unordered list format, which is accessible to assistive technologies.

2. Good color contrast:

The color scheme maintains sufficient contrast between text and background for readability by users with visual impairments.

Compliance

Sometimes I had issues with font sizes looking too small or too big on different devices. I solved this by using relative units like em and rem instead of fixed pixel sizes, so the text adjusted better on different screen sizes. I also had trouble with images not resizing properly. They would either overflow the container or look stretched. I fixed this by using max-width: 100% and height: auto in CSS to keep the images responsive. Another challenge was making sure elements stayed in the right position when the screen size changed. I used CSS Flexbox and Grid to create a more flexible layout and added media queries to adjust styles for smaller screens. Sometimes, the colors and background didn’t contrast well, making text hard to read. I fixed this by choosing better color combinations and making sure there was enough contrast for accessibility.
